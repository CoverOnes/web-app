專案描述

本案為將既有 EDA (Electronic Design Automation) 工具的單體式 API Gateway 重構為基於 Kubernetes 的微服務架構。原系統承載每日約 1.2 億次 內部請求，目前面臨擴展性瓶頸與部署困難。

期望接案團隊具備「大型 monolith 拆分實戰經驗」，能在不停機的前提下，逐步將原系統拆分為 8-12 個獨立微服務，並導入 Service Mesh、分散式追蹤與灰度發布機制。

本專案屬於台積電子核心研發部門使用之內部工具，需簽署 NDA 後始可獲得詳細技術文件。
